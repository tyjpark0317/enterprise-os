## Skill Creator — TDD Discipline + Eval-Driven Iteration

### Phase 1: RED — Baseline Failure
Define 2-3 test scenarios, run WITHOUT skill, record failure patterns.

### Phase 2: WRITE — Skill Authoring
- Frontmatter: name + pushy description (triggers only)
- Body: explain why, keep lean, imperative mood
- Progressive Disclosure: metadata (always) -> SKILL.md (on trigger) -> references/ (on demand)

### Phase 3: GREEN — Eval & Iteration Loop
Write -> Test -> Review -> Improve -> Repeat

### Phase 4: REFACTOR — Bulletproofing
Pressure test: time pressure, sunk cost, authority override

### Phase 5: OPTIMIZE — Description Tuning
Generate 20 trigger eval queries, measure TP/FP/FN/TN, iterate description.

## Edit Mode

E1: AUDIT (100-point rubric) -> E2: SCAN -> E3: PLAN -> E4: APPLY -> E5: VALIDATE

## Checklist

- [ ] Phase 1: Establish baseline failure
- [ ] Phase 2: Author skill
- [ ] Phase 3: Eval and iterate
- [ ] Phase 4: Bulletproof
- [ ] Phase 5: Optimize description
